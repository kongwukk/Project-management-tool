# 智能项目助手 - 项目描述文档

## 1. 项目目标
构建一个桌面应用程序，通过自然语言交互辅助管理基于Markdown文件的项目。用户可上传Office文档、配置多种大模型API，程序会自动读取项目目录下的所有 `.md` 文件作为上下文，使模型回答能紧扣项目实际状态。

## 2. 功能模块划分

### 2.1 主界面 (UI)
- **布局**：
  - **顶部栏**：模型下拉选择器 + “配置API”按钮 + 项目路径显示/刷新按钮。
  - **中央区域**：对话消息列表（用户与助手的交替气泡），支持滚动。
  - **底部区域**：多行文本输入框 + “上传文件”按钮 + “发送”按钮。
- **交互**：
  - 输入框支持回车发送（Shift+Enter换行）。
  - 上传文件后，在输入框上方显示已加载文件列表（可移除）。
  - 对话中每条消息显示时间戳。

### 2.2 模型管理模块
- **功能**：
  - 管理多个模型配置（名称、API地址、密钥、默认参数）。
  - 支持切换当前使用的模型。
  - 提供配置弹窗，可新增、编辑、删除、测试连接。
- **存储**：配置加密保存于本地（如 `config.json`）。

### 2.3 文件处理模块
- **支持格式**：
  - Word (.docx) → 提取段落文本、表格。
  - PowerPoint (.pptx) → 提取每页幻灯片中的文本（包括备注）。
  - Excel (.xlsx) → 提取所有工作表的数据为表格形式（CSV风格）。
- **输出**：结构化文本，附带文件名和来源类型，供对话引擎使用。

### 2.4 项目上下文管理
- **职责**：
  - 监听或定期扫描配置的 `projects/` 目录。
  - 读取所有 `.md` 文件内容，解析标题、正文、任务列表、表格等。
  - 将内容合并为统一的文本上下文（可设置前缀，如“当前项目状态如下：”）。
  - 支持手动刷新，或利用文件系统监听（如 `chokidar`）自动更新。
- **上下文注入方式**：在每次对话请求时，将最新上下文作为系统提示（System Prompt）或用户消息前缀。

### 2.5 对话引擎
- **输入组装**：
  - 系统提示（包含项目上下文）
  - 历史对话（可配置窗口大小）
  - 当前用户输入
  - 附加文件内容（若已上传）
- **调用模型**：通过统一的API适配器，支持OpenAI兼容格式，可扩展至其他厂商。
- **响应处理**：支持流式输出（逐字显示）或一次性返回，错误处理（重试、提示）。

### 2.6 配置持久化
- 存储内容：模型配置列表、当前选中模型、项目目录路径、UI主题等。
- 格式：JSON，位于用户数据目录（如 `~/.project-assistant/config.json`）。

### 2.7 日志与记录
- 自动保存每次对话会话（按时间命名），存储为Markdown文件，便于回顾。
- 可选导出功能。
